# 项目协作约定

英文版见 [AGENTS.en.md](AGENTS.en.md)。

最后更新：2026-03-13

## 目标

本文件用于约束本仓库后续的代码注释、文档语言和更新方式，避免文档风格来回切换。

## 规则

### 1. 代码注释使用英文

- 所有代码内注释都使用英文
- 包括：
  - 行内注释
  - 块注释
  - 函数说明注释
  - 配置文件中的注释
- 不要求把变量名、函数名专门改成中英文混合，保持现有工程风格即可

### 2. 面向阅读的 Markdown 默认使用中文

凡是为了让人阅读、理解、协作而写的 Markdown，默认中文优先。

包括但不限于：

- `README`
- `progress`
- `todo`
- `explore`
- `vendor` 说明
- 各类运行说明、设计说明、调研记录

也就是说，仓库里直接给人看的主文档，默认应当是中文版本。

### 3. 每次更新文档时，同时维护中英文两份

只要某个 Markdown 文档是面向阅读的，就应当同时存在：

- 一份中文版本
- 一份英文版本

推荐命名规则：

- 中文主文件：
  - `README.md`
  - `progress.md`
  - `todo-loop.md`
- 英文副本：
  - `README.en.md`
  - `progress.en.md`
  - `todo-loop.en.md`

目录内的 `README` 也遵循同样规则：

- `vendor/README.md`
- `vendor/README.en.md`

### 4. 中文版本是默认入口

- 中文版本保留主文件名
- 英文版本使用 `*.en.md`
- 当用户直接打开仓库文档时，应优先看到中文内容

### 5. 更新要求

之后凡是发生以下行为，都应同步维护双语版本：

- 新增可读性文档
- 修改已有说明文档
- 更新进度记录
- 更新 TODO
- 补充调研笔记

如果某次更新只改了中文，没有改英文，视为文档更新不完整。

## 当前执行方式

本仓库从现在开始采用以下约定：

- 代码注释：英文
- 可读 Markdown：中文主文件 + 英文副本
- 中文为默认阅读入口

## 备注

这个约定主要作用于“给人读”的内容，不要求把程序输出、第三方官方引用、协议字段名强行翻译成中文。
