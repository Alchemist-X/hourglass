# 录制检查清单

## 开始前

- [ ] 打开 https://hourglass-eta.vercel.app 确认加载正常
- [ ] 所有 Section 滚动流畅，无白屏/加载失败
- [ ] 终端准备好 `pnpm ave:demo` 命令（不要回车）
- [ ] 终端字体大小调大（确保录制后可读，推荐 16pt+）
- [ ] 浏览器缩放 100%，全屏
- [ ] 关闭无关浏览器标签页
- [ ] 关闭通知/勿扰模式（macOS: 控制中心 → 专注模式）
- [ ] 录制软件就绪（推荐 OBS 或 QuickTime）
- [ ] 分辨率 1920x1080
- [ ] 网络稳定（录制前 ping 一下网站）
- [ ] 如用旁白，确认麦克风正常

## 录制顺序

1. [ ] 打开 `title-card.html`，录 3 秒
2. [ ] 切到网站 Hero（沙漏插画 + 金色标题），停 2 秒后缓慢向下滚动（共 40 秒）
3. [ ] 经过 3 个统计指标卡牌 → 停在"问题与方案"对比区
4. [ ] 停在 Boss Encounter + 4 张 Skill 卡牌区（50 秒）
   - 每张卡牌停留 8-10 秒
   - 指向能量球（左上角得分）和信号条（底部渐变条）
   - 指出底部的 AVE API 端点标签
5. [ ] 停在信号聚合 + Edge 大数字区域（50 秒）
   - 重点停在 "我们的概率: 21%" vs "市场赔率: 3.1%"
   - Edge = 17.9% 大数字上停 3 秒
6. [ ] 停在交易执行 + 风控区域（40 秒）
   - 依次指向 6 个风控检查项
   - 强调"服务层硬风控"概念
7. [ ] 停在交易结果 + 思考过程时间轴（40 秒）
   - 指向交易记录表格
   - 重点停在"总耗时 2.1 秒"
8. [ ] 切到终端，运行 `pnpm ave:demo`（30 秒）
   - 等待彩色输出完整展示
   - 上下滚动展示关键段落
9. [ ] 切回网站架构图区域（30 秒）
   - 从下往上指向每一层
   - 强调 Layer 1（AVE Claw）和 Layer 3（风控）
10. [ ] 缓慢滚回 Hero 页面，停 3 秒，结束录制

## 备选方案

- 如果网站加载慢 → 准备截图/录屏做备选素材
- 如果不配旁白 → 用字幕替代（导入 SRT 文件）
- 如果终端 Demo 出错 → 可跳过该段，多展示网站
- 推荐多录几遍，第一遍熟悉流程，第二遍正式录制

## 重点提示

1. **节奏** — 不要赶。让评审看清每个画面。每张 Skill 卡牌至少停留 8 秒。
2. **重点时间** — 花最多时间在 Skill 卡牌分析和真实交易结果，这两段最打动评审。
3. **评审视角** — 评审最关心：你用 AVE 做了什么？有什么实际价值？代码能跑吗？
