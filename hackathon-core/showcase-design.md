# Hourglass 黑客松展示页面设计文档

> 目标：评审打开 URL 后，5 分钟内完全理解项目价值并留下深刻印象
> 形式：单页滚动，从叙事到 Demo 到结果，一气呵成
> 美学：**Slay the Spire × 量化交易** — 暗色质感、卡牌式信息架构、羊皮纸纹理、金属边框、策略博弈感
> 核心视觉母题：**沙漏 (Hourglass)** — 贯穿品牌 logo、加载动画、时间进度、能量流动
> 最后更新：2026-04-14

---

## 一、设计原则

1. **叙事驱动** — 页面从上到下讲一个完整故事，不是堆功能
   - 问题 → 方案 → 原理 → 实证 → 结果
2. **单市场聚焦** — 用一个具体市场（BTC $150K）贯穿全页，提供切换按钮
3. **推理透明** — 每一步分析都可视化，评审能看到 AI 怎么想
4. **数据真实** — 展示真实交易记录和持仓，不是纯 mock
5. **视觉优先** — 先看图再看字，卡牌和信号球比文字更有说服力
6. **卡牌隐喻** — 每个 AVE Skill 是一张"卡牌"，有边框、能量球、效果描述，交易就是"打牌"
7. **秒级响应，持久化运营** — 不是一次性分析工具，是持续运转的自主交易 Agent

## 二、色彩系统

### 设计灵感

Slay the Spire 的视觉语言：暗色背景营造沉浸感，金色边框赋予卡牌仪式感，红/蓝/绿语义色区分攻击/技能/能力。
我们将这套语言映射到量化交易：暗色 = 专业交易终端，金色 = 价值与边际，红/绿 = 看跌/看涨。

### 品牌色（沙漏主题）

| 名称 | 色值 | 用途 |
|------|------|------|
| 沙漏金 (Primary) | #d4a574 | 品牌 logo、卡牌边框、高亮文字、沙漏图标 |
| 深邃蓝 (Secondary) | #5fa8d3 | 稀有卡牌边框、链接、交互元素、特殊高亮 |
| 暗夜紫 (Accent) | #7b2d8e | 警告状态、debuff 效果、异常检测 |

- Hero 区域的沙漏插画（沙粒 = 数据流，上半 = AVE 信号，下半 = 交易执行）

## 四、核心视觉组件

### 4.1 AVE Skill 卡牌（核心组件）

每个 AVE Skill 的分析结果渲染为一张 Slay the Spire 风格的卡牌。卡牌类型色参考 STS：

| 类型 | STS 对应 | 边框色 | 含义 |
|------|---------|--------|------|
| 信号牌 (Signal) | Attack | #e63946 红 | 直接产生交易信号的分析 |
| 数据牌 (Data) | Skill | #2a9d8f 绿 | 提供防御性/信息性数据 |
| 能力牌 (Power) | Power | #5fa8d3 蓝 | 持续生效的监控能力 |

**卡牌结构（自上而下）：**

```
┌─────────────────────────────────┐
│  ╔═══════════════════════════╗  │  ← 外层：金色(#d4a574)金属边框，2px，圆角8px
│  ║ ⬡+0.73  KLINE ANALYSIS   ║  │  ← 顶栏：能量球(左上) + 卡牌名(居中，Cinzel)
│  ║ ─────── Signal Card ───── ║  │  ← 类型条：红色底条 + 白字 "Signal Card"
│  ║                           ║  │
│  ║  ┌───────────────────┐    ║  │  ← 卡牌画面区：占卡牌 40% 高度
│  ║  │                   │    ║  │     放置迷你 K 线图 / 鲸鱼柱状图
│  ║  │  [数据可视化区域]  │    ║  │     真实绘制的图表，非占位符
│  ║  │                   │    ║  │
│  ║  └───────────────────┘    ║  │
│  ║                           ║  │
│  ║  MA20 > MA50    ✦ Bullish ║  │  ← 效果描述区：Inter 正文
│  ║  MACD: 金叉              ║  │     关键数据 + 简明判断
│  ║  波动率: 3.2%/日          ║  │
│  ║                           ║  │
│  ║  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░   ║  │  ← 信号条：渐变填充，左红右绿
│  ║  Signal: BULLISH (+0.73)  ║  │     当前值用三角标记
│  ╚═══════════════════════════╝  │
└─────────────────────────────────┘
```

**能量球设计（Signal Score Orb）：**

```
     ╭──────╮
    │  ⬡     │   ← 圆形球体，径向渐变
    │ +0.73  │   ← 中心数值，JetBrains Mono, bold
     ╰──────╯
      glow ↑       ← 外发光：正值绿光，负值红光，零值灰光
```

- 球体大小：40x40px
- 位于卡牌左上角（模仿 STS 的 mana cost 位置）
- 数值范围 [-1.00, +1.00]
- 正值：绿色渐变球 (#2a9d8f → #0e3d38) + 绿色外发光
- 负值：红色渐变球 (#e63946 → #4a0e14) + 红色外发光
- 零值：灰色渐变球 (#8b8b9e → #4a4a5e)

### 4.2 信号管线地图（STS Act Map 风格）

取代线性流程图，使用分支路径地图展示数据流向，模仿 STS 的关卡地图：

```
                    ☆ START
                    │
              ╔═════╪═════╗
              ║     │     ║
         [📊 Price]  [📈 K-line]
         Data Card   Signal Card
              ║     │     ║
              ╚═════╪═════╝
                    │
              ╔═════╪═════╗
              ║     │     ║
         [🐋 Whale]  [📉 Ratio]
         Signal Card  Data Card
              ║     │     ║
              ╚═════╪═════╝
                    │
               ═══[⚡]═══
               BOSS: EDGE
               Edge: +17.9%
                    │
               [🎯 TRADE]
               Execute Order
```

地图节点用圆形图标，连线用虚线动画（数据流动感），当前激活节点发光。
Boss 节点（Edge 计算）用更大的菱形，发金色光芒。

### 4.3 生命值条（Portfolio Health / HP Bar）

模仿 STS 的 HP 条设计：

```
❤️ HP: ████████████████████░░░░░  $17.94 / $20.00  (89.7%)
       ←── 当前净值 ──→ ←─ 已亏 ─→

🛡️ Block: ████████████████████████████████████████  Drawdown: 0%
          ←── 30% 止损阈值尚远 ──→
```

- HP 条：红色底 (#e63946)，当前值用绿色填充 (#2a9d8f)
- 低于 70% 时条变黄 (#f4a261)，低于 50% 时闪烁红色
- 数字叠在条上方，白色投影文字
- Block 条（护甲）：蓝色 (#5fa8d3)，代表风控缓冲

### 4.4 遗物栏（Risk Controls = Relics）

STS 中遗物是被动效果，始终显示在屏幕底部。我们的风控规则同样是"被动 buff"，永远生效：

```
┌─────────────────────────────────────────────────────────────────┐
│  RELICS (风控规则 · 始终生效)                                    │
│                                                                 │
│  🛡️ 单笔上限    🛡️ 总敞口     🛡️ 止损线     🛡️ 最大持仓        │
│     15%           80%          30%           22               │
│  MAX_TRADE     EXPOSURE    DRAWDOWN     MAX_POS              │
│                                                                 │
│  🛡️ 单事件上限   🛡️ 最低交易                                     │
│     30%           $5                                           │
│  EVENT_EXP     MIN_TRADE                                      │
└─────────────────────────────────────────────────────────────────┘
```

每个遗物是一个小图标（40x40px），hover 时弹出详细说明。
图标用金色边框圆形，内部放盾牌 emoji，底部显示数值。
风控通过时图标发绿光，被触发时发红光 + 震动动画。

### 4.5 状态效果（Active Positions = Buffs/Debuffs）

STS 中角色可以有 buff（正面）和 debuff（负面）效果。持仓就是状态效果：

```
  盈利持仓 (Buffs):                    亏损持仓 (Debuffs):
  ┌──────┐ ┌──────┐                    ┌──────┐
  │ 📈   │ │ 📈   │                    │ 📉   │
  │ +12% │ │ +5%  │                    │ -3%  │
  │ BTC  │ │ ETH  │                    │ SOL  │
  └──────┘ └──────┘                    └──────┘
  绿色边框 + 绿色发光                   红色边框 + 红色发光
```

#### 3b: 四张 AVE Skill 卡牌（2x2 卡牌手牌展开）

像 STS 战斗中的手牌展开，4 张卡牌扇形排列或 2x2 网格。
每张卡片是一个独立的分析卡牌，对应 `CryptoSignal` 接口的不同字段。

数据对应关系：
- 卡牌 1 对应 `price` 字段
- 卡牌 2 对应 `trendScore` + `details.klines`（KlineDetails: ma20, ma50, macdSignal, volatility）
- 卡牌 3 对应 `whalePressure` + `details.whales`（WhaleDetails: buyVolume, sellVolume, netRatio, largeTradeCount）
- 卡牌 4 对应 `sentimentScore` + `details.sentiment`（SentimentDetails: buy5m, sell5m, buy1h, sell1h, buy6h, sell6h）

## 七、数据来源

所有展示数据来自两个来源：
1. **AVE Mock/Real** -- `CryptoSignal` 数据（价格、K线、鲸鱼、买卖比），结构定义在 `ave-crypto-signals.ts`
2. **Polymarket Real** -- 真实持仓和交易记录（通过 spectator 模式）

页面服务端渲染，数据在 build 时或请求时生成，不依赖客户端 API 调用。

### 7.1 核心数据接口

**CryptoSignal（from ave-crypto-signals.ts）：**
```typescript
interface CryptoSignal {
  token: string           // "BTC" | "ETH"
  tokenId: string         // AVE token contract address
  price: number           // current price
  trendScore: number      // K-line trend score [-1, +1]
  whalePressure: number   // whale pressure [-1, +1]
  sentimentScore: number  // buy/sell sentiment [-1, +1]
  overallScore: number    // composite score [-1, +1]
  details: {
    klines: { ma20, ma50, macdSignal, volatility }
    whales: { buyVolume, sellVolume, netRatio, largeTradeCount }
    sentiment: { buy5m, sell5m, buy1h, sell1h, buy6h, sell6h }
  }
  timestamp: string
}
```

**ProbabilityEstimate（from ave-signal-to-probability.ts）：**
```typescript
interface ProbabilityEstimate {
  marketQuestion: string
  token: string
  targetPrice: number
  targetDirection: "above" | "below" | "hit"
  currentPrice: number
  aveScore: number                 // = CryptoSignal.overallScore
  estimatedProbability: number     // our probability estimate [0.05, 0.95]
  marketImpliedProbability: number // Polymarket current odds
  edge: number                    // = estimated - market
  confidence: number              // confidence level [0, 1]
}
```

### 7.2 信号聚合权重

来源：`ave-crypto-signals.ts` 的 `BASE_WEIGHTS` 常量

| 信号组件 | 权重 | 失败降级 |
|---------|------|---------|
| trendScore (K线) | 0.4 | 该组件归零，其余按比例放大 |
| whalePressure (鲸鱼) | 0.3 | 同上 |
| sentimentScore (买卖比) | 0.3 | 同上 |

当某组件 AVE 调用失败时，`reweightForFailures()` 函数自动将剩余组件权重按比例归一化。

### 7.3 概率估算逻辑

来源：`ave-signal-to-probability.ts` 的 `estimateProbability()`

1. **基础概率** -- 基于价格距离 + 剩余时间
   - 当前价 >= 目标价: base = 0.8
   - 差距 <= 10%: base = 0.5
   - 差距 10-30%: base = 0.35
   - 差距 > 30%: base = 0.2
   - 时间调整：剩余天数越多，base 越靠近 0.5

2. **AVE 调整** -- `aveScore * 0.15`（最大 15% 的概率偏移）

3. **最终概率** -- `clamp(baseProbability + aveAdjustment, 0.05, 0.95)`

4. **Edge** -- `estimatedProbability - marketImpliedProbability`

### 7.4 实际下单思考过程（以 BTC $150K 仓位为例）

基于当前真实持仓，展示完整的 AI 思考链和下单顺序：

| 步骤 | 阶段 | 耗时 | 关键数据 |
|------|------|------|---------|
| 1 | 信号采集 | 0.8s | 4 个 AVE API 并行调用 |
| 2 | 信号聚合 | <0.1s | overallScore = +0.505 |
| 3 | 概率估算 | <0.1s | 21% vs 市场 3.1% → Edge +17.9% |
| 4 | 风控检查 | <0.1s | 6 项遗物检查 (5 PASS, 1 WARN) |
| 5 | 执行下单 | 1.0s | FOK, Type 2 签名, 5 shares @ $0.031 |
| 6 | 持仓记录 | <0.1s | 加入监控 + Auto-Research 持续跟踪 |

总耗时 ~2.1 秒，体现"秒级响应"。此后 Agent 持续运行，每 5 分钟重新评估，持续优化——体现"持久化运营"。
