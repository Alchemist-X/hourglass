# Hourglass Demo 视频脚本

> 总时长：≤ 5 分钟
> 录制方式：屏幕录制 + 旁白（或字幕）
> 分辨率：1920x1080 或更高
> 核心目标：让评审在 5 分钟内理解 "我们用 AVE 链上数据在预测市场找到了别人看不到的 edge"

## 详细脚本

### 0:00 - 0:40 | 开场：项目介绍（40秒）

**画面：** 打开 https://hourglass-eta.vercel.app ，展示 Hero 页面（沙漏插画 + 暗色背景 + 金色标题）

**旁白/字幕：**

> "Hourglass 是一个链上信号驱动的预测市场交易 Agent。
>
> 大多数预测市场参与者依赖新闻和直觉做判断。但链上数据——鲸鱼的买卖行为、实时价格趋势、交易量异常——这些信号比新闻更早、更真实。
>
> Hourglass 利用 AVE Claw 的 4 个 Skill 模块，7x24 小时自动监控链上数据，在 Polymarket 预测市场上发现定价偏差，自动执行交易。"

**操作：**
1. 页面加载完成后停留 2 秒，让评审看到整体视觉风格
2. 缓慢向下滚动，经过沙漏插画和 3 个统计指标卡牌（4 AVE Skills / 7 目标市场 / 真实交易）
3. 在 "问题与方案" 对比卡牌区域短暂停留

---

### 0:40 - 1:30 | AVE Skill 集成展示（50秒）

**画面：** 滚动到 "市场遭遇"（Boss Encounter）和 4 张 Skill 卡牌区域

**旁白/字幕：**

> "我们选定了这个市场：'Bitcoin 会在 2026 年 6 月 30 日前达到 $150K 吗？'交易量 310 万美元，当前赔率只有 3.1%。
>
> 接下来看 Hourglass 的 AI 是怎么分析的——通过 4 个 AVE Claw Skill，每一个都是一张'卡牌'：
>
> 第一，📊 实时价格——通过 AVE 的 tokens/price API 获取 BTC 当前价格 $94,200，距离目标 $150K 需要涨 59.2%。这个数据本身不产生方向信号，但它是所有分析的基础。
>
> 第二，📈 K线趋势分析——AVE 的 klines API 返回多周期 K 线数据。MA20 站上 MA50，MACD 金叉确认，趋势看涨，得分 +0.73。
>
> 第三，🐋 鲸鱼追踪——AVE 的 txs API 监控链上大额交易。过去 1 小时买入 4230 万、卖出 1810 万，净买入 +2420 万，47 笔大额交易。
>
> 第四，📉 买卖比——AVE 的 token detail API 显示 5 分钟买卖比 1.86 倍、1 小时 1.59 倍，链上情绪偏多。"

**操作：**
1. 先在 Boss Encounter 卡牌（市场信息）停留 3 秒，指向交易量和赔率
2. 依次指向每张 Skill 卡牌，每张停留 8-10 秒
3. 每张卡牌指向能量球（左上角得分）和信号条（底部渐变条）
4. 注意指出底部的 AVE API 端点标签

---

### 1:30 - 2:20 | 信号聚合 + Edge 计算（50秒）

**画面：** 滚动到信号聚合区域（4-Skill 信号瀑布 + Edge 大数字）

**旁白/字幕：**

> "四个信号综合后——K 线权重 40%，鲸鱼 30%，买卖比 30%——得到综合得分 +0.52，看涨。
>
> 基于这个综合得分和价格距离，AI 估算 BTC 达到 $150K 的概率是 21%。
>
> 而 Polymarket 的赔率只给了 3.1%。
>
> 这意味着我们发现了 17.9 个百分点的 Edge——市场严重低估了这个概率。
>
> 这就是链上数据的信息优势：鲸鱼在大量买入、技术指标看涨、链上情绪偏多——这些信号都指向同一个方向，但大多数预测市场参与者只看新闻，看不到这些。"

**操作：**
1. 指向信号聚合的权重分配展示
2. 停留在 "我们的概率: 21%" vs "市场赔率: 3.1%" 的对比处
3. 重点停在 Edge = 17.9% 的大数字上，停留 3 秒
4. 如果有信号瀑布动画，让它完整播放一次

---

### 2:20 - 3:00 | 交易执行 + 风控（40秒）

**画面：** 滚动到交易执行和风控区域

**旁白/字幕：**

> "发现 Edge 后，Hourglass 自动执行交易：买入 5 shares YES，价格 $0.031 每 share，使用 Polymarket 的免 Gas EIP-712 签名。
>
> 但下单前必须通过 6 层服务层风控检查：
> - 单笔不超过 15% 资金
> - 总敞口不超过 80%
> - 单事件不超过 30%
> - 最大 22 个并行持仓
> - 回撤超过 30% 自动停机
> - 最低交易额 $5
>
> 所有检查通过后才执行。这不是提示词级别的风控——是服务层硬编码的规则，AI 无法绕过。"

**操作：**
1. 指向交易执行详情（买入方向、数量、价格）
2. 依次指向 6 个风控检查项，每个旁边应有绿色通过标记
3. 强调"服务层硬风控"的概念

---

### 3:00 - 3:40 | 真实交易结果（40秒）

**画面：** 滚动到真实交易记录 + 思考过程时间轴

**旁白/字幕：**

> "这是真实的链上交易，不是模拟。
>
> 钱包 0xc788 在 Polymarket 上持有两个活跃仓位——都是 AI 自动分析、自动下单的结果。每一笔交易都可以在链上验证。
>
> 整个推理过程只用了 2.1 秒——从 AVE 价格获取，到 K 线分析，到鲸鱼追踪，到买卖比，到信号聚合，到风控检查，到下单完成。
>
> 而且 Hourglass 每 5 分钟重新运行一次。它不是一次性的分析工具，而是一个 7x24 小时持久化运营的自主 Agent。50 次以上的 Pulse 分析运行记录证明了系统的稳定性。"

**操作：**
1. 指向交易记录表格（市场名称、方向、金额、状态）
2. 指向思考过程时间轴，展示每一步的时间戳和耗时
3. 重点停在 "总耗时 2.1 秒" 上
4. 如有持仓 PnL 展示，短暂指向

### 3:40 - 4:10 | 终端演示（30秒）

**画面：** 切换到终端，运行 `pnpm ave:demo`

**旁白/字幕：**

> "在终端也能看到完整的推理过程：
>
> 信号采集 → 聚合 → 市场匹配 → Edge 计算 → 风控检查 → 执行。
>
> 每一步都有具体的数据和分数，完全透明可追溯。彩色输出区分 INFO、WARN、ERR、OK 不同级别。"

**操作：**
1. 在终端输入 `pnpm ave:demo` 并回车
2. 等待彩色输出完整展示（约 10 秒）
3. 上下滚动，展示关键输出段落
4. 如果终端输出很长，重点展示信号聚合和 Edge 计算部分

2. 强调 Layer 1（AVE Claw）和 Layer 3（风控）
3. 如果有代码结构展示，快速指向 monorepo 子包列表

## 录制准备清单

### 录制前检查
- [ ] 网站 https://hourglass-eta.vercel.app 加载正常
- [ ] 所有 Section 滚动流畅，无白屏/加载失败
- [ ] `pnpm ave:demo` 在终端可正常运行
- [ ] 网络稳定（录制前 ping 一下网站）
- [ ] 屏幕分辨率 1920x1080 或更高
- [ ] 录制软件准备好（OBS / QuickTime / Loom）

### 录制环境
- [ ] 关闭无关浏览器标签页
- [ ] 关闭通知（勿扰模式）
- [ ] 终端字体大小调大（确保录制后可读）
- [ ] 浏览器缩放 100%
- [ ] 如用旁白，确认麦克风正常

### 备选方案
- [ ] 如果网站加载慢，准备截图/录屏做备选素材
- [ ] 如果不配旁白，用字幕替代（推荐使用剪映/CapCut 加字幕）
- [ ] 如果终端 Demo 出错，可跳过该段，多展示网站

## 重点提示

1. **节奏** — 不要赶。让评审看清每个画面。每张 Skill 卡牌至少停留 8 秒。
2. **重点时间分配** — 花最多时间在 Section 2（AVE Skill 卡牌分析）和 Section 4（真实交易结果），这两段最能打动评审。
3. **评审视角** — 评审最关心三个问题：
   - "你用 AVE 做了什么？"（Section 2 回答）
   - "这有什么实际价值？"（Section 3-4 回答）
   - "代码能跑吗？"（Section 5 回答）
4. **视频上传** — 建议上传到 YouTube（unlisted）或腾讯视频，确保评审能流畅观看。
5. **多录几遍** — 第一遍熟悉流程，第二遍正式录制，选最好的一版提交。
